# Refactoring Prompt

## Objective

Improve the current implementation without changing its behavior.

---

## Goals

Improve:

- readability
- maintainability
- architecture
- naming
- separation of concerns

---

## Constraints

Do not:

- change functionality
- introduce unnecessary abstractions
- violate project architecture

---

## Output

Explain every refactoring.

Explain why it improves the code.
